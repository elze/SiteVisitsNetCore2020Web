Page visits

For more explanation of this app, go to Home.

It is build from a skeleton ASP.NET Core / WebAPI / Angular 8 app that Visual Studio 2019 creates for you, and it still has the two component VS creates: Counter and FetchData. I didn't delete them because when everything breaks, they provide me a useful reference to determine what level things are broken at. This is a reverse chronological list of page visits. In this view, each visit contains only the basic information, hence we call them flat visits. You may notice that Page URL and Referrer (or Came From) fields display the information as concisely as possible. For example, Page URL leaves out the domain name, because I already know it. I'm only interested in the remaining page path. Similarly, Came From, wherever possible, fishes out just the relevant nuggets from the referrer string (if a visitor came to the page from a search engine, those strings can be extremely long and cumbersome to display).

You can page through the visits by clicking on the paginator arrows at the bottom of the page, but also by editing the URL directly, for example, http://elzecode.com/visitsnc3ng/flat-visits/2/size/25
